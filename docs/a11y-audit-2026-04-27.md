## 3. 修正方針

並列 10 仮想エージェントによる同時着手で、概ね 1 セッション内に完遂する戦略を採る。

| Agent | 担当領域 | 主な作業 |
| --- | --- | --- |
| **A1** | `header.tsx` | dropdown `aria-controls` / mobile button `aria-expanded` 付与 |
| **A2** | `editor.tsx` | `role="dialog"` / `aria-modal` / Esc handler / フォーカストラップ / icon button `aria-label` ×4 |
| **A3** | `sections.tsx` | 装飾番号 `aria-hidden` / `<summary>` focus-visible / h2 共通コンポーネント化 |
| **A4** | `layout.tsx` | Skip link to main content 追加 |
| **A5** | `globals.css` | universal `:focus-visible` scaffold + `prefers-reduced-motion` メディアクエリ |
| **A6** | `footer.tsx` | リンク hover/focus 微調整 (合格済みだが磨き込み) |
| **A7** | E2E | Playwright によるキーボードナビゲーション試験スクリプト追加 |
| **A8** | docs | 本 a11y 監査レポートの記述 |
| **A9** | 本番 | `/admin` 配下の本番管理画面でキーボード単独操作試験 |
| **A10** | policy | `docs/a11y-policy.md` への運用ポリシー策定 |

### 3.1 並列実行のメリット

- ファイルが互いに独立しているためコンフリクトリスクなし
- 1 ファイル 1 エージェントで責務分離 → レビューが容易
- 全タスク完了後 A7 (E2E) と A9 (本番試験) で統合検証

## 4. 新規追加した a11y 機能

| # | 機能 | 実装ファイル | 効果 |
| --- | --- | --- | --- |
| 1 | **Skip link to main content** | `layout.tsx` | キーボードユーザが先頭で `Tab` キー一回でメイン領域へジャンプ可能 |
| 2 | **Universal `:focus-visible` scaffold** | `globals.css` | すべての操作可能要素にフォーカスリング統一適用 |
| 3 | **`prefers-reduced-motion` 対応** | `globals.css` | 前庭障害ユーザのモーション酔い防止、`transition` / `animation` を瞬間化 |
| 4 | **Theme-color meta tag (light/dark)** | `layout.tsx` | iOS Safari / Android Chrome のステータスバー色がテーマと連動 |
| 5 | **Heading scroll-margin-top** | `globals.css` | アンカーリンク遷移時に固定ヘッダで隠れない |
| 6 | **Lang attribute on English text** | `sections.tsx` 他 | SR が英語部分を英語音声で正しく読み上げ |

### 4.1 実装スニペット参考

#### Skip link (layout.tsx)

```tsx
<a href="#main" className="sr-only focus:not-sr-only ...">
  メインコンテンツへスキップ
</a>
```

#### Reduced motion (globals.css)

```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```
